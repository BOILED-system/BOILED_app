# 清算・集金管理機能 設計計画書

## 1. 概要
本機能は、サークルの会計担当（各ジャンルリーダー含む）が関わる複雑な集金業務（参加費、衣装代、スタジオ代など）を効率化し、誰が払って誰が未払いなのかを一元管理するシステムである。「会員番号」認証における決済の透明性を担保するため、最終的な承認は必ず外部名義との突合にて行う。

## 2. ゴール
*   幹部（Admin）が清算項目を作成し、指定した対象者（ジャンル/ナンバー等）に対して請求できるようになること。
*   メンバー（Member）がアプリ上で対象の集金項目を確認し、支払いを行った旨を報告できること。
*   幹部側で実際の口座・PayPay着金履歴とアプリの報告を突合し、ステータスを「支払い済（消込完了）」に変更できること。

## 3. 機能の詳細設計（フロントエンド）

### 3.1. 清算の新規作成 (`/settlements/new`, `/events/[id]`)
*   **イベント紐付け**:
    *   イベント詳細ページから直接作成する場合、`eventId` を保持。
*   **対象者の指定**:
    *   イベント作成同様、`targetGenreIds`（ジャンルIDの配列）で指定する。
    *   もしイベントに紐づく清算であれば、デフォルトでイベントの対象ジャンルを引き継ぐ。
*   **入力項目**:
    *   タイトル（例: 「M1衣装代」）
    *   金額（例: 5000）
    *   支払期限（カレンダーピッカー等）
    *   銀行振込先口座情報（テキストエリア）
    *   PayPay送金先情報（リンクURLやテキスト）

### 3.2. 支払い実行画面・一覧 (`/payments`, `/events/[id]`)
*   **一覧表示（Member）**:
    *   自身に関連する清算（未払い・支払報告済み・消し込み完了）を一覧表示する。
    *   期限が近いもの、期限切れのものを警告ビューで目立たせる。
*   **支払い報告モーダル**:
    *   「PayPayで送金しました」か「銀行振込しました」かを選択し、「支払いを完了したとして報告」ボタンを押す。
    *   押下後、ステータスは `PAID_REPORTED`（支払い報告済み、幹部確認待ち）となる。

### 3.3. 幹部向け 消し込み画面 (`/admin/payments` 等の新設)
*   現在幹部向けの支払い一覧管理・消し込み画面が存在しないため新規開発。
*   **UI構成**:
    *   清算項目ごとのドリルダウン（例：「M1衣装代」を選択）。
    *   メンバー一覧とステータス（未払い / 報告済み / 消込完了）を表示。
    *   報告済みのユーザーに対し、「着金確認OK（CONFIRMED）」ボタンを押してステータスを確定する。

## 4. 機能の詳細設計（バックエンド / API）

### 4.1. データモデルの拡張
*   **`settlements` コレクション**:
    *   `id`: string
    *   `title`: string
    *   `amount`: number
    *   `dueAt`: timestamp
    *   `bankInfo`: string
    *   `paypayInfo`: string
    *   `targetGenreIds`: string[] (**NEW** - ジャンル機能導入に伴い `targetUserIds` から移行)

### 4.2. 清算・支払いステータスの遷移
*   **`UNPAID` (未払い)**: デフォルト。清算対象のメンバーがまだ何も操作していない状態。
*   **`PAID_REPORTED` (報告済/確認待ち)**: メンバーが「送金した」と報告したが、会計の着金確認が終わっていない状態。
*   **`CONFIRMED` (消込完了)**: 会計が実際の口座を確認し、システム上で消し込みを行った状態。

### 4.3. バックエンドAPIの主な改修ポイント
*   **`POST /api/settlements`**:
    *   清算の作成。ペイロードに `targetGenreIds` を受け取り保存。
    *   現状は `targetUserIds`（ユーザーID配列）を受け取っている箇所をジャンル機能に合わせる。
*   **`GET /api/settlements/me`**:
    *   リクエスト元のユーザーが所属するジャンル（`genreId`）が含まれる `settlements` を抽出し、そのユーザーの `payments`（支払い状況）をJOINして返却する。
*   **`PUT /api/admin/payments/:id/confirm` (新規)**:
    *   Admin権限専用API。メンバーの支払い報告（`PAID_REPORTED`）を `CONFIRMED` に昇格させるロジックを実装する。
