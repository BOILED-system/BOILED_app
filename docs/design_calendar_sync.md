# カレンダー連携機能 設計計画書

## 1. 概要
本機能は、アプリ上で作成・登録されたイベント情報（全体練習や深夜練など）のうち、**メンバー自身が参加対象となっているイベント**を個人のGoogleカレンダー（または共通の共有カレンダー等）へ自動的にエクスポート（同期）する仕組みを提供する。これにより、「アプリを開かなくても自分のスケジュールが把握できる」という利便性を実現する。

## 2. ゴール
*   メンバーがアプリ内で「自分の関係するイベント」をGoogleカレンダーで確認できること。
*   イベントの作成、日時変更、削除があった際、カレンダー上にもその変更が適宜反映されること。

## 3. 同期方式の検討

同期には大きく分けて2つのアプローチが存在し、運用負荷やサークルのリテラシーに応じて選択する。今回は、**方式A（iCal購読型）**を基本とし、より強固な運用を求める場合は**方式B（Google Calendar API型）**を検討する。

### 方式A: iCal形式購読による連携（推奨・MVP向け）
*   **概要**: バックエンドAPI（Go）で、各ユーザー個別の `.ics`（iCalendar形式）ファイルを動的に生成するURLを発行し、ユーザーはそれをGoogleカレンダーやiOSカレンダーから「照会（サブスクライブ）」する。
*   **メリット**: 実装が非常に容易。GoogleアカウントでのOAuth認証などのややこしい実装が不要。
*   **デメリット**: Googleカレンダーの仕様上、更新頻度が最大24時間に1回程度となるため、直前のスケジュール変更が同期されにくい。

### 方式B: Google Calendar APIによる直接書き込み
*   **概要**: アプリ（またはサービスアカウント）を通じて、メンバー指定のカレンダーへイベントを直接 `POST / PUT / DELETE` する。
*   **メリット**: 変更が瞬時に反映されるため、直前の深夜練の場所変更などにも強い。
*   **デメリット**: ユーザー全員に「Googleアカウントでのログイン・Googleカレンダーへのアクセス許可（OAuth）」を強要する必要があり、今回の「会員番号のみの簡易ログイン」方針とコンフリクトする。

**📝 今回の採用方針**:
「会員番号のみでのログイン」方針に基づくため、OAuthを伴う方式Bの導入は見送る。
したがって本プロジェクトでは **方式A（iCal購読型）** を正として設計する。

## 4. 機能の詳細設計（バックエンド / API）

### 4.1. iCalフィード配信API
*   **`GET /api/users/:memberId/calendar.ics`**:
    *   **処理**:
        1.  指定された `memberId` のユーザーが所属するジャンル（`genreId`一覧）を取得。
        2.  `targetGenreIds` が空（全体）、または該当ジャンルが含まれる直近のイベント（`events` / `practice_series`）をFirestoreから抽出。
        3.  抽出したイベントデータを、標準的な iCalendar フォーマット (`text/calendar` メディアタイプ) の文字列に変換してレスポンスとして返す。
    *   **セキュリティ考慮**:
        カレンダーURLが他人に推測されるのを防ぐため、`memberId` の後ろに推測困難なランダムトークン（例: `A001_x9f8h2jf9`）を付与し、そのURLでのみ購読可能とする。

### 4.2. iCalデータの生成内容
*   **`BEGIN:VEVENT`**: イベント1件につき1ブロック。
*   **`DTSTART`, `DTEND`**: イベントの開始・終了日時（JSTをUTCに変換して出力、または `TZID=Asia/Tokyo` 指定）。
*   **`SUMMARY`**: イベント名（例: `【全体】新歓公演リハーサル`, `【M1】深夜練`）。
*   **`LOCATION`**: スタジオ名や場所。
*   **`DESCRIPTION`**: イベントのURL（アプリの当該イベントページへのリンク）や備考を記載。
*   **`UID`**: Firestoreの `eventId` をそのまま利用し、イベント変更時にカレンダー上で上書き更新されるようにする。

## 5. 機能の詳細設計（フロントエンド）

### 5.1. カレンダー連携設定画面 (`/profile` 等)
*   **UI構成**:
    *   「カレンダーと同期する」セクションを設ける。
    *   ユーザー固有のカレンダー購読用URL（`https://api.../users/A001_xxx/calendar.ics`）を表示。
    *   「URLをコピー」ボタン、および「Googleカレンダーに追加する」等のショートカットボタンを設置。
*   **同期の手順（ユーザー向けガイド）**:
    画面上に「コピーしたURLをGoogleカレンダーの『URLで追加』から登録してください」という短い説明文を添える。
