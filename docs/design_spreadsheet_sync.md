# スプレッドシート連携機能 設計計画書

## 1. 概要
本機能は、ダンスサークルの各ジャンルリーダー（幹部）が全体およびナンバーごとの出欠状況（出席、欠席、遅刻、早退、メモ）を一覧で俯瞰し、集計・管理できるようにするため、アプリ上の出欠データ（RSVP）をリアルタイム または 定期バッチでGoogleスプレッドシートへ自動出力する仕組みを提供する。

## 2. ゴール
*   幹部がアプリの管理画面ではなく、普段使い慣れているGoogleスプレッドシート上で全員の出欠状況を確認できること。
*   イベントの作成や出欠の変更があった際、自動的にシートへ反映されること。
*   ジャンルごと、あるいは月ごとのタブ（シート）に分割して出力する等の拡張性を持たせること。

## 3. アーキテクチャ構成案

Firestoreのドキュメント作成・更新の変更イベントをトリガーとして、Google Sheets APIを直接叩くか、Google Apps Script（GAS）のWeb APIを経由してスプレッドシートへ書き込む。今回は拡張性や幹部側のメンテナンス性を考慮し、**GAS経由での連携**を推奨する。

*   **アプリ側（Go Backend）**:
    *   定期バッチ（Cron）でデータをまとめて送信、またはイベント/出欠更新APIが叩かれたタイミングでデータを整形して送信。
*   **GAS (Google Apps Script) 側**:
    *   `doPost(e)` 関数を利用してWebアプリとして公開。
    *   受け取ったJSONデータをパースし、指定されたスプレッドシートへ書き込む。

## 4. 機能の詳細設計（GAS側）

### 4.1. データ受け取り API (`doPost`)
*   **エンドポイント**: GASでデプロイしたWebアプリケーションのURL
*   **メソッド**: `POST`
*   **ペイロードフォーマット例**:
    ```json
    {
      "type": "event_rsvp_update",
      "event": {
        "id": "event_123",
        "title": "深夜練（M1）",
        "date": "2026-04-10T23:00:00Z"
      },
      "rsvps": [
        { "memberId": "A001", "name": "山田太郎", "status": "GO", "note": "" },
        { "memberId": "A002", "name": "鈴木花子", "status": "LATE", "note": "24時から" }
      ]
    }
    ```

### 4.2. スプレッドシート側の処理ロジック
1.  **シートの特定**:
    送信された `event.id` や `title` から、対象となるイベント専用のシート（タブ）がすでに存在するかチェックする。なければ新規作成する。
2.  **シートの更新**:
    受け取った `rsvps` のリストを走査し、シート上の行を更新・または追記する。
    *   A列: 会員番号 (`memberId`)
    *   B列: 氏名 (`name`)
    *   C列: 出欠ステータス (出/欠/遅/早)
    *   D列: メモ
    *   E列: 最終更新日時
3.  **集計機能の保持**:
    シート上部（1行目など）に関数（`=COUNTIF(...)`等）を配置しておき、「出席〇人、欠席〇人」などの情報が自動で計算される状態を保つ。

## 5. 機能の詳細設計（バックエンド / Go API）

### 5.1. 連携トリガーの実装
*   **リアルタイム連携方式（推奨）**:
    `POST /api/events/:eventId/rsvp`（出欠登録/変更処理）がユーザーから叩かれ、Firestoreの更新が成功した直後に、非同期（goroutine等）で対象イベントの全RSVPを取得し、まとめてGASのURLへPOST送信する。
*   **定期バッチ方式**:
    Cloud Scheduler等を使用して、1日1回（例えば夜中など）に未連携・または更新があったイベントのRSVPデータをまとめてGASへ送信する。リアルタイム性が求められない場合はこちらの方がサーバー負荷が低い。

### 5.2. 送信データの生成
*   DBに保存されている `userId` から、ユーザーの名前 (`name`) と会員番号 (`memberId`) をJOIN（DB参照）して補完した上で、連携用JSONを作成する。
*   ※送信先（GASのURL）は環境変数 `GAS_WEBHOOK_URL` などに定義してセキュアに管理する。

## 6. ユースケース / 運用フロー

1.  **イベント作成時**: Adminが新しく「新歓公演リハーサル」というイベントを作成する。
2.  **シート自動作成**: GASへ連携処理が走り、対象スプレッドシートに「新歓公演リハーサル」というタブが自動生成される。
3.  **出欠入力**: 各Memberがアプリから出欠（GO, NO, ...）とメモを入力する。
4.  **シート自動更新**: 非同期でGASへデータが送られ、「新歓公演リハーサル」シートの指定メンバーの行が出席・欠席に書き換わる。
5.  **幹部確認**: ジャンルリーダーはアプリの管理画面を開かなくとも、スマホのGoogleスプレッドシートアプリから全員の最新の出欠状況とメモ（遅刻時間など）を一覧で確認できる。
